# SPECIFICATION — make-comment-movie

## 1. 目的

画面録画 mp4 と WhoWatch 形式のコメント TSV から、**左に黒パネルを追加した解像度**の動画を生成し、パネル上に**コメントを焼き込む**（ソフト字幕ではなく映像の一部として出力する）。

## 2. 入力

### 2.0 補助シェル（mp4 だけ指定）

リポジトリの `scripts/make-comment-movie-from-mp4.sh` は、**1 本以上の mp4**（同一ディレクトリに置いたパスを列挙）を渡すと、各ファイルごとに**同一ディレクトリ**の `<ベース名>.comments.txt` をコメント源とみなし、**`<ベース名>.with-comments.mp4`** に出力する。引数列は呼び出し元シェルが **グロブ等で展開した結果**（例: `録画*.mp4` → 複数パス）をそのまま受け取る。`make-comment-movie` 本体の CLI 引数（例: `--video-start`）は、**先頭の連続する存在する `.mp4` パス列の直後**から透過的に付与する（`--` で明示的に区切ることも可）。**複数本を処理する場合も同じ引数**が各実行に使われる。バイナリは **`MAKE_COMMENT_MOVIE` 環境変数**、**`target/release/make-comment-movie`**、**`PATH` の `make-comment-movie`** の順で解決する。いずれか 1 本が失敗したら**終了コード非 0**（途中まで成果物ができている場合もありうる）。

### 2.1 動画（mp4）

- ffmpeg がデコードできる形式（主に H.264 + AAC の mp4 を想定）。
- **動画の時刻 0 秒**に対応する**絶対時刻（ローカル）**を、次のいずれかで与える。
  - **ファイル名**に `YYYY-MM-DDTHH-MM-SS` を含む（正規表現で最初の一致を使用）。
  - **`--video-start`** 文字列（`YYYY-MM-DDTHH:MM:SS` または `YYYY-MM-DD HH:MM:SS`、秒の小数は任意）。

タイムゾーンは**コメントログの `posted_at_local` と同一の解釈**（ローカル日時の差分）を前提とする。

### 2.2 コメントファイル（WhoWatch ログ）

- 先頭に `#` で始まるメタ行があってよい。
- ヘッダ行: `posted_at_local` で始まる 1 行をスキップする。
- データ行: **タブ区切り**。列は少なくとも次の順を想定する。

| 列番号（0 始まり） | 名前 | 用途 |
|-------------------|------|------|
| 0 | posted_at_local | 表示同期の基準時刻 |
| 1 | user_id | （本ツールでは未使用） |
| 2 | display_name | 画面上の表示名 |
| 3 | account_name | （本ツールでは未使用） |
| 4 | comment_type | フィルタ（`--skip-playitem` 時に `BY_PLAYITEM` を除外） |
| 5 | message | 本文 |

列は **6 本**（タブ区切り）。旧版で誤って 7 列前提になっていた不具合を修正済み。

`posted_at_local` の形式: `YYYY-MM-DD HH:MM:SS` または小数秒付き（例: `.000`）。

## 3. 時刻同期

- `video_start`: 動画 t=0 の `NaiveDateTime`。
- 各行の `posted_at` を `NaiveDateTime` にパースし、  
  `start_sec = (posted_at - video_start)` を**秒（実数）**で求める。
- **動画開始より前**のコメント（`posted_at < video_start`）は**捨てる**。
- **同一 `start_sec`** になる行は、表示のため **1ms 単位で後ろの行をずらす**（ゼロ長 ASS セグメント防止）。
- フィールド `end_sec` は次コメントの `start_sec` を格納する（互換・デバッグ用）。ASS の区間長には用いない。

## 4. 映像処理

1. **ffprobe** で最初の映像ストリームの幅 `W_vid`・高さ `H_vid`、および **`format.duration`**（秒）を取得する。
2. 左パネル幅 `W_panel` は `--panel-width`、または既定:  
   `clamp(round(H_vid * 0.30), 200, 560)`。
3. 出力解像度: `W_out = W_panel + W_vid`, `H_out = H_vid`。
4. **vf チェーン**（概念）:
   - `pad=W_out:H_out:W_panel:0:black` … 左に黒、元映像を右へ `x=W_panel` に配置。
   - `subtitles=ASSパス` … libass で ASS を焼き込み。任意で `fontsdir` を付与。

映像コーデック: **libx264**（CRF・preset は CLI）。音声: **copy** を試みる。

## 5. 字幕（ASS）

- `PlayResX = W_out`, `PlayResY = H_vid`。
- スタイル: **上揃え左（Alignment=7）**、本文は白字・黒縁。表示名は **`\\c`** で明るめグリーン（ASS `&H00B4FF7F`）とし、本文前に白へ戻す。**MarginL / MarginR** で論理テキスト幅を定める。
- **手動折り返し**: Unicode Line Breaking（`unicode-linebreak` の Mandatory / Allowed）で区切り候補を列挙し、`unicode-width` で表示幅を積算して **px 換算の列上限**を超える直前に分割する。行区切りは **`\N` のみ**（日本語は語区切り、欧文はスペース等を優先）。
- **[Script Info] `WrapStyle: 2`**（単語折り返しオフ）とイベント先頭の **`\\q2`**（自動改行オフ）で、手動 `\N` のみ有効にする。**`\\q3` は libass 上で別モードとなり自動折り返しと競合しうるため使用しない。**
- 各コメントは表示名に緑・本文に白の **`\\c`** を挟み、**`{\\fs小\\c緑}表示名\\N{\\fs大\\c白}本文`** 形式。表示名と本文の間は改行（`\\N`）。リスナー名は本文より小さい `\\fs`（`--name-font-size` / `--font-size`）。
- **水平余白（非対称）**: 左は **`PANEL_PAD_LEFT_PX`（既定 10px）** で `MarginL` / `\\pos`。映像側は **`PANEL_PAD_RIGHT_PX`（既定 1px）** だけ空け、`MarginR = PlayResX - W_panel + PANEL_PAD_RIGHT_PX` とする。折り返し用の有効幅は **`W_panel - 左 - 右`** で、右の空きを減らしつつ左は広めに保つ。outline は 1。
- **表示セグメント**（ギャップなし）: 同時刻コメントはまとめ、各セグメントは **`[t_k, t_{k+1})`**（最後は **`[t_last, duration]`**）。`t` は動画内秒、`duration` は ffprobe の尺。
- 各セグメントに **Dialogue を 1 本**だけ出し、**次のコメントが来るまで**同じスタックを表示し続ける（途中で字幕が消えない）。
- 本文は、そのセグメント代表時刻において  
  `j <= 代表インデックス` かつ `代表時刻 - start_j <= --max-dwell-sec` の行を選び、**新しいコメントから順**に `\N` で連結する（画面上は **上が最新**、下へいくほど古い）。**`--max-lines` 件を超える場合は時系列で新しい側だけ残す**。
- **Alignment=7**（上揃え左）とパネル上端付近の `\pos` で、ブロックが下方向へ伸びると既存表示が下へ押し出される見え方になる。
- 新セグメント開始時、可能なら **`\\move`** でブロックをわずかに上から下へ落とす（`--scroll-ms`、セグメントが短い場合は `\pos` のみ）。

## 6. 制限・既知の注意

- **コメントが 0 件**（全行が動画開始前、または `--skip-playitem` で全除外）の場合はエラー終了する。
- 同一秒内に大量コメントがあっても **セグメントはマージ**され、**件数は `max_lines`** で打ち切る。**古すぎる件**は `--max-dwell-sec` でスタックから落ちる。
- 折り返しは生成側で行うため libass の自動折り返しには依存しない。1 行の **unicode 列上限**は、パネル内の利用可能幅（px）とフォントサイズから **`≈ 2 * (幅 / フォントサイズ)`**（全角幅 2 を前提）に近づけ、実フォントの字詰まりに合わせ **`COLS_FILL_RATIO`（既定 1.30）** と**切り上げ（`ceil`）**で上振れさせ、右の空きを抑える。ごく稀に右端がわずかにクリップする場合は `--panel-width` か比率を下げる。
- フォント・fontconfig 環境により、既定フォント名が解決できない場合がある。**`--font-name` / `--fonts-dir`** で調整すること。
- **欠グリフ**: コメントに CJK 以外のレアブロック（例: エジプト象形文字 U+13000 台、一部絵文字）が含まれると、ffmpeg の `subtitles`（libass）が **`Glyph 0x…. not found`** を返すことがある。これは **主フォント＋探索パス上のフォールバックにそのコードポイントが無い**ためであり、処理が即失敗するとは限らないが当該字は表示されない。**`--fonts-dir` に当該ブロックをカバーするフォントを追加**するか、OS に同種フォントをインストールして対応する。
- 音声 `copy` が失敗するコンテナ・コーデックの組み合わせでは、ffmpeg 手動での音声エンコードが必要になることがある（本ツールは現状オプション未実装）。

## 7. 依存外部コマンド

- `ffmpeg`
